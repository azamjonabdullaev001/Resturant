# 🍽️ Ресторан — Система управления

Полная система управления рестораном с панелями для разных ролей.

## Панели

| Панель | Описание |
|--------|----------|
| **Админ-панель** | Дашборд, управление сотрудниками, столами, категориями, меню, заказами |
| **Официант** | Просмотр меню, создание заказов, отслеживание статуса |
| **Шашлычник** | Входящие заказы на шашлык, управление своими блюдами |
| **Самсачик** | Входящие заказы на самсу, управление блюдами |
| **Нац. блюда** | Входящие заказы на национальные блюда |
| **Десерты** | Управление десертами и напитками |

## Технологии

- **Frontend**: React + Vite
- **Backend**: Go (Gin)
- **БД**: PostgreSQL
- **Контейнеризация**: Docker Compose

## Запуск

```bash
docker-compose up --build
```

После запуска:
- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:8080

## Вход в систему

### Администратор (по умолчанию)
- **Телефон**: `+998914751330`
- **Пароль**: `12345678`

### Другие пользователи
Создаются через админ-панель → Сотрудники → Добавить.

## Рабочий процесс

1. **Админ** добавляет столы, сотрудников и категории
2. **Кухня** (шашлычники, самсачики, нац. блюда) добавляют свои блюда с фото и ценами
3. **Клиент** садится за стол
4. **Официант** выбирает стол → добавляет блюда в корзину → отправляет заказ
5. Заказ автоматически распределяется по кухонным панелям
6. **Кухня** видит заказ, начинает готовить, отмечает "Готово"
7. **Админ** видит все заказы, отмечает оплату

## API Endpoints

| Метод | URL | Описание |
|-------|-----|----------|
| POST | /api/login | Авторизация |
| GET | /api/me | Текущий пользователь |
| GET | /api/dashboard | Статистика (админ) |
| CRUD | /api/users | Сотрудники (админ) |
| CRUD | /api/tables | Столы |
| CRUD | /api/categories | Категории |
| CRUD | /api/menu | Меню |
| GET/POST | /api/orders | Заказы |
| PUT | /api/orders/:id/pay | Оплата заказа |
| GET | /api/panel/:type/orders | Заказы для кухни |
| PUT | /api/order-items/:id/status | Обновить статус блюда |
| POST | /api/upload | Загрузка фото |
